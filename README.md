# 🐲 Monster PVP — الموقع + بوت تذاكر الديسكورد (نسخة مُصححة)

## ✅ ما الذي تم تصحيحه

1. **`style.css` المفقود** — تم إنشاؤه بالكامل (الموقع كان بدون أي تصميم).
2. **توحيد الكود** — تم تنظيم المشروع في مجلدات: `public/` (الموقع)، `routes/` (API)، `models/` (قاعدة البيانات)، `config/` (الاتصال بـ MongoDB).
3. **حذف ملف `toast.js` المكرر** — الدالة موجودة بالفعل داخل `script.js`.
4. **تصحيح تضارب `CLIENT_ID`** — كان مختلفاً بين `.env` و `script.js`. الآن الواجهة تجلب القيمة من السيرفر عبر `/api/config` فتطابق `.env` دائماً.
5. **🚨 إزالة Discord Webhook المكشوف من `checkout.html`** — كان يعمل من المتصفح مباشرة، أي زائر يقدر يشوفه ويستخدمه.
6. **🚨 إعادة تنظيم `.env`** — كل القيم (التوكن، الـ IDs، الرولات، إعدادات قاعدة البيانات...) في مكان واحد مع شرح كل متغير.
7. **إضافة قاعدة بيانات MongoDB Atlas** — التذاكر، التقييمات، وعدّاد التذاكر تُحفظ بشكل دائم (لم تعد تضيع عند إعادة تشغيل البوت).
8. **كل شيء يعمل محلياً (Local)** — لا يوجد أي Hosting أو خدمة خارجية. الموقع والبوت يعملان من نفس السيرفر على جهازك.

## ⚠️ تحذيرات أمنية مهمة (نفّذها فوراً)

كان التوكن والـ Webhook **مكشوفين بالكامل** في الكود القديم. اعتبرهم مخترقين:

1. اذهب إلى **Discord Developer Portal** → تطبيقك → **Bot** → اضغط **Reset Token** → ضع التوكن الجديد في `.env`.
2. اذهب لروم الديسكورد الذي كان فيه الـ Webhook القديم (`...1514204164622782588...`) → Integrations → Webhooks → **احذف ذلك الويبهوك**.

## 📦 خطوات التشغيل المحلي

### 1) تثبيت الحزم
```bash
npm install
```

### 2) إعداد ملف `.env`
الملف `.env` جاهز وفيه القيم القديمة (التوكن القديم — **يجب تجديده** كما بالأعلى).
راجع `.env.example` لشرح كل متغير.

### 3) إنشاء قاعدة بيانات MongoDB Atlas (مجانية)
1. سجّل حساب في https://www.mongodb.com/cloud/atlas/register
2. أنشئ **Cluster مجاني** (M0 Free Tier).
3. من **Database Access** → أنشئ مستخدم (Username/Password).
4. من **Network Access** → أضف IP الخاص بجهازك، أو `0.0.0.0/0` للسماح من أي مكان (مناسب للتشغيل المحلي/التجربة).
5. من **Database → Connect → Drivers** → اختر Node.js → نسخ رابط الاتصال:
   ```
   mongodb+srv://USERNAME:PASSWORD@cluster0.xxxxx.mongodb.net/monster_pvp?retryWrites=true&w=majority
   ```
6. ضع هذا الرابط في `.env` بجانب `MONGODB_URI=`.

### 4) تسجيل أوامر السلاش (مرة واحدة فقط، أو عند تغيير الأوامر)
```bash
npm run deploy
```

### 5) تشغيل المشروع (الموقع + البوت معاً)
```bash
npm start
```

سيظهر:
```
✅ تم الاتصال بقاعدة بيانات MongoDB Atlas بنجاح
🌐 الموقع وسيرفر الربط مع البوت يعملان على: http://localhost:3000
✅ Logged in successfully as: ...
```

### 6) فتح الموقع
افتح المتصفح على:
```
http://localhost:3000
```

## 📌 ملاحظات إضافية

- **الصور والفيديو غير موجودة** في الملف الأصلي (`Plus.jpg`, `Clown.jpg`, `Fight.jpg`, `Girl.jpg`, `ID.jpg`, `Monster.mp4`). الكود يشير إليها لكنها غير مرفوعة. ضعها داخل مجلد `public/` بنفس الأسماء حتى تظهر في الموقع (وإلا ستظهر الصورة مكسورة فقط، والباقي سيعمل بشكل طبيعي).
- جميع الـ IDs (الرولات، القنوات، الأقسام) أصبحت في `.env` — لو غيّرت أي رول أو قناة في سيرفرك، فقط حدّث القيمة في `.env` بدون لمس الكود.
- `DISCORD_WEBHOOK_URL` تُرك فاضي عمداً لأن الإرسال المباشر من المتصفح غير آمن. كل التوثيق الآن يتم عبر البوت + قاعدة البيانات.
